Sparkle helps development teams use AI coding tools with more confidence.

It brings lightweight, project-aware guidance into your local workflow so developers can stay aligned with team standards while moving fast.

## What you can do with Sparkle

With Sparkle, you can:

- Use the Sparkle extension inside supported IDEs
- Run Sparkle from the command line
- Connect your workspace
- Apply project guidance during development
- Keep your local workflow simple and repeatable

Sparkle is designed to fit into how developers already work. You do not need to change your entire development process to get started.

## How Sparkle is typically used

Most teams start with one of two workflows:

### IDE workflow

Use the Sparkle extension inside your editor to access Sparkle while you code.

This is the recommended starting point for most developers.

### CLI workflow

Use the Sparkle CLI to run Sparkle commands from your terminal.

This is useful for local workflows, scripts, and repeatable project setup.

## Before you begin

You will need:

- Access to a Sparkle workspace
- A supported IDE or terminal environment
- Permission to connect your local project
- A project repository you want to use with Sparkle
